# 监控仪表板修复报告
## 完成时间：2026年3月4日 00:30

## 📋 问题描述
用户反馈 `https://lakeli.top/monitor_dashboard.html` 地址打不开。

## 🔍 问题分析
经过分析，发现以下问题：

### 1. Nginx配置问题
- 生产环境Nginx配置将所有请求都代理到后端FastAPI服务（`http://127.0.0.1:8000`）
- 缺少对静态HTML文件`monitor_dashboard.html`的直接服务配置
- 监控仪表板文件无法通过Nginx直接访问

### 2. API地址配置问题
- 监控仪表板中的API地址硬编码为特定IP和端口
- 无法根据当前访问域名自动适配API地址

### 3. 文件部署问题
- 监控仪表板文件未部署到Nginx的静态文件目录
- 缺少自动化部署脚本

## 🔧 修复方案实施

### 1. 修改监控仪表板API地址配置
**文件**: `monitor_dashboard.html`
**修改内容**:
```javascript
// 修改前
const API_BASE = (() => {
    if (window.location.hostname === 'localhost' || window.location.hostname === '127.0.0.1') {
        return 'http://localhost:8001/api/v1';
    }
    return 'http://120.26.201.61:8001/api/v1';
})();

// 修改后
const API_BASE = (() => {
    const baseUrl = window.location.origin;
    console.log('当前域名:', baseUrl);
    
    if (window.location.hostname === 'localhost' || window.location.hostname === '127.0.0.1') {
        return 'http://localhost:8001/api/v1';
    }
    
    return baseUrl + '/api/v1';
})();
```

### 2. 修改Nginx配置支持静态文件
**文件**: `nginx/conf.d/prod.conf`
**添加内容**:
```nginx
# 监控仪表板静态文件
location = /monitor_dashboard.html {
    # 假设监控仪表板文件放在 /usr/share/nginx/html 目录下
    root /usr/share/nginx/html;
    try_files $uri $uri/ =404;
    add_header Cache-Control "no-cache, no-store, must-revalidate";
    add_header Pragma "no-cache";
    add_header Expires "0";
}
```

### 3. 创建自动化部署脚本
**文件**: `scripts/deploy-monitor-dashboard.sh`
**功能**:
- 检查本地文件是否存在
- 上传监控仪表板文件到阿里云服务器
- 上传Nginx配置文件到阿里云服务器
- 重启Nginx服务并验证配置
- 测试监控仪表板访问

## 🔄 故障排除

### 问题1：404 Not Found
**可能原因**：
- 监控仪表板文件未上传到正确位置
- Nginx配置中的root路径不正确
- 文件权限问题

**解决方案**：
```bash

# 检查Nginx配置

# 检查Nginx错误日志
tail -f /var/log/nginx/prod-error.log
```

### 问题2：502 Bad Gateway
**可能原因**：
- 后端API服务未运行
- Nginx代理配置错误

**解决方案**：
```bash
# 检查后端服务状态
systemctl status docker  # 如果使用Docker
docker ps  # 查看容器状态

# 检查API服务是否响应
curl http://127.0.0.1:8000/health
```

### 问题3：API请求失败
**可能原因**：
- 监控仪表板中的API地址配置错误
- CORS问题
- 后端API服务异常

**解决方案**：
1. 打开浏览器开发者工具，查看Console和Network标签
2. 检查API请求的URL是否正确
3. 检查后端API服务日志
