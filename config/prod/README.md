# 生产环境配置说明

## 配置管理原则
生产环境配置**不存储在本地代码库中**，遵循以下原则：
- 本地只维护开发环境配置 (`config/dev/`)
- 生产配置直接维护在阿里云服务器上
- 敏感信息（密码、密钥）不入代码库

## 生产配置位置
- **服务器**: 阿里云ECS (120.26.201.61)
- **项目目录**: `/root/project/prod/zhuangxiu-agent/`
- **配置目录**: `config/prod/`
- **主要文件**: `docker-compose.prod.yml`

## 配置内容特点
生产环境配置包含：
- 阿里云服务器的绝对路径
- 生产数据库连接信息
- Redis缓存配置
- 其他敏感环境变量

## 开发与部署流程
1. **本地开发**: 使用 `config/dev/` 配置进行开发测试
2. **代码提交**: 测试通过后提交到git仓库
3. **服务器更新**: 在阿里云服务器拉取最新代码
4. **生产部署**: 使用服务器上的生产配置部署

## 配置变更管理
- 生产配置变更直接在阿里云服务器上进行
- 重要变更应记录在 `docs/阿里云部署指南.md`
- 配置备份由服务器运维负责

## 注意事项
- ❌ 不要将生产配置提交到git
- ❌ 不要将敏感信息放入代码库
- ✅ 本地只关注开发环境配置
- ✅ 部署时使用服务器上的生产配置
